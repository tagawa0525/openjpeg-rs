# Phase 700: HTJ2K（High-Throughput JPEG 2000）

Status: PLANNED

## Context

Phase 600（JP2 + 公開API）完了後の次フェーズ。ITU-T T.814に基づくHTJ2Kデコーダを実装する。既存のT1デコーダ（MQ算術符号）と並列に、高スループットモードのコードブロック復号を提供する。

Phase 700が完了すると、HT-JPEG 2000（Part 15）でエンコードされたファイルのデコードが可能になる。

## HTJ2Kコードブロック構造

```text
コードブロックビットストリーム:
  ├─ MagSgn（前方読み）   — マグニチュード・サイン符号
  ├─ MEL + VLC（後方読み）— ランレングス + 可変長符号
  └─ SPP + MRP（後方読み）— 伝播パス + 精密化パス
```

3つのコーディングパス:

1. **Cleanup Pass (CUP)**: クワッド（2×2）単位でシグニフィカンスパターン復号
2. **Significance Propagation Pass (SPP)**: 新規シグニフィカントサンプル復号
3. **Magnitude Refinement Pass (MRP)**: 係数マグニチュード精密化

## 設計判断

### モジュール配置

`coding/ht_dec.rs`に配置。既存の`coding/t1.rs`から`cblksty & HT`フラグで分岐呼び出し。

### ビットストリームリーダー

C版の`dec_mel_t`, `rev_struct_t`, `frwd_struct_t`をRust構造体として実装。ポインタ演算は配列インデックスに置換。

### ルックアップテーブル

C版の`t1_ht_luts.h`（事前計算済み配列）をそのまま`static`定数として`ht_luts.rs`に配置。生成ツール(`t1_ht_generate_luts.c`)は不要。

### スコープ外（延期）

| 項目               | 延期先     | 理由                   |
| ------------------ | ---------- | ---------------------- |
| HTJ2Kエンコード    | 将来       | Part 15は復号優先      |
| マルチスレッド     | Phase 800  | feature flag           |
| SIMD最適化         | Phase 900  | パフォーマンスフェーズ |
